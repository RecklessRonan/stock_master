# 股票调研深度提升计划

## 现状诊断

当前系统的调研信息不足主要集中在三层：

```
数据层 → context.md → LLM 多角色分析
  ↑ 信息瓶颈在此    ↑ 质量上限受制于context
```

### 数据层现有覆盖

- 基本信息（东财）
- 日K线 + 技术指标（MA/MACD/RSI/KDJ/布林带）
- 财务摘要（仅年报损益表，5年）
- 估值（PE/PB/PS 当日绝对值，无历史分位）
- 近期新闻（标题列表）

### 数据层缺失维度（信息盲区）

- **估值历史分位**：PE/PB 当前在历史哪个百分位？便宜还是贵？（`stock_a_indicator_lg` 已有全量历史数据，但只取了最后一行）
- **现金流**：经营/投资/筹资现金流完全缺失（只有损益表）
- **季报数据**：只有年报，季度层面的景气趋势感知缺位
- **资金流向**：主力/机构净流入、北向资金动态 完全没有
- **股权结构动态**：前十大股东变化、大股东增减持 完全没有
- **融资融券**：融资余额及其变化（市场情绪重要指标）
- **重大公告/事件**：定增、回购、并购、业绩预告 完全没有
- **同行横向对比**：估值、增速、盈利能力与同行的量化对比 完全缺失

### 分析层缺陷（五维评分严重失真）

查看 `[src/stock_master/analysis/quantitative.py](src/stock_master/analysis/quantitative.py)`：

- `score_growth`（成长性）：实际是**价格动量**（60日涨幅 + MA20位置），与营收/利润增长无关
- `score_profitability`（盈利能力）：完全用 **PE 代替**，PE低≠盈利能力强（低PE可能是衰退股）
- `score_safety`（安全性）：只看价格波动率和回撤，无财务安全性指标（负债率、流动比率）

### 研究提示层不足

- 5个 Prompt 模板的 `{{context}}` 输入质量有限，LLM 分析上限受制于 context
- `03-risk.md` 没有量化红旗指标（Z-Score、连续三年现金流/利润背离等）
- `05-industry.md` 的可比估值表纯靠 LLM 推理，没有真实数据
- 所有提示均缺少**宏观/政策背景**注入

## 改进方案

### 方向一：扩充 fetcher.py 数据维度

新增以下数据采集函数（均使用现有 AkShare）：

- `fetch_valuation_history(code)` — 拉取1-3年历史PE/PB序列，计算当前分位数
  - A股: `ak.stock_a_indicator_lg` 已含历史数据，只需扩展窗口
- `fetch_cashflow(code)` — 年度现金流量表
  - A股: `ak.stock_cash_flow_sheet_by_annual_em`
  - 港股: `ak.stock_financial_hk_cash_em`
- `fetch_quarterly_financial(code)` — 最近4-6个季度财务摘要
  - A股: `ak.stock_financial_abstract_ths(indicator="按单季度")`
- `fetch_fund_flow(code)` — 个股资金流向（主力净流入）
  - A股: `ak.stock_individual_fund_flow`
- `fetch_holder_changes(code)` — 前十大股东及持仓变化
  - A股: `ak.stock_main_fund_flow` + `ak.stock_holder_top10_em`
- `fetch_major_announcements(code)` — 重大公告列表
  - A股: `ak.stock_notice_report` 或 `ak.stock_bulletin_em`

### 方向二：修复五维量化评分

修改 `[src/stock_master/analysis/quantitative.py](src/stock_master/analysis/quantitative.py)`，新增参数 `financial: pd.DataFrame`：

- `score_growth` → 改为：营收YoY增速 + 利润YoY增速（财务驱动，保留价格动量作为辅助）
- `score_profitability` → 改为：ROE + 毛利率 + 净利率（从财务摘要取值）
- `score_safety` → 增加：资产负债率 + 流动比率（财务 + 价格波动复合）

### 方向三：丰富 context.md 内容

修改 `[src/stock_master/analysis/reporter.py](src/stock_master/analysis/reporter.py)` 和 `[src/stock_master/pipeline/context_builder.py](src/stock_master/pipeline/context_builder.py)`，新增以下章节：

- **估值历史分位** — 当前PE/PB处于近3年XX%分位，附简单解读
- **现金流摘要** — 近3年经营/投资/筹资现金流及自由现金流
- **季度财务趋势** — 近4季度营收/利润趋势（判断加速/减速）
- **资金流向摘要** — 近5/10/20日主力净流入及近期趋势
- **股权动态** — 最新前十大股东，标注近期变化

### 方向四：增强研究提示模板

- `**03-risk.md`**：增加要求分析师检查现金流/利润背离、商誉减值风险、存货/应收款异动等量化红旗
- `**05-industry.md`**：基于 context 中的同行数据进行真实对比，而非凭空推理
- **新增 `00-macro-context.md`**：注入宏观背景模板（当前利率环境、相关政策、行业景气度），作为其他4个角色的前置公共上下文

## 执行顺序

优先级排序（信息增量/开发成本比）：

1. 估值历史分位（改动最小，信息增量最大）
2. 修复五维评分（逻辑错误，必须修）
3. 现金流 + 季报数据（财务分析核心缺失）
4. 资金流向摘要（增加市场情绪维度）
5. 股权动态（增加持仓变化洞察）
6. 重大公告（增加事件驱动感知）
7. 增强研究提示模板（最后调整）
