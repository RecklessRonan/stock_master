# 股票调研信息补全计划

## 当前核心问题诊断

通过对 `context.md` 实际产出（09988/09999/00100 等）与 Prompt 框架的对比，发现存在以下系统性缺口：

### 问题 1：港股 K 线数据大规模缺失

- 所有港股样本（09988/09999/00100）的 K 线均显示"暂无 K 线数据"
- 直接导致成长性、安全性、动量三个维度全部默认 50.0（无效分数）
- 技术面快照完全为 N/A，技术面 Prompt (`04-technical.md`) 形同虚设

### 问题 2：量化评分与基本面严重脱节

- "成长性"用价格涨幅，不是业务收入增长
- "盈利能力"只看 PE 数值，不是 ROE/利润率
- `score_profitability` 函数注释写"ROE 和利润率"，实现是纯 PE 判断

### 问题 3：缺少 Prompt 框架要求但实际未拉取的数据


| Prompt 要求                     | 现状        |
| ----------------------------- | --------- |
| 同行可比估值表（`05-industry.md`）     | 无任何可比公司数据 |
| PE/PB 历史分位（`02-financial.md`） | 只有当前时点值   |
| ROIC、自由现金流（`02-financial.md`） | 未拉取       |
| 大股东减持、融资融券（`03-risk.md`）      | 未拉取       |
| 资金流向（北向/南向）（`03-risk.md`）     | 未拉取       |
| 分析师评级与目标价                     | 完全缺失      |
| 股东结构与外资持股                     | 完全缺失      |
| 公告与重大事项                       | 完全缺失      |


### 问题 4：新闻质量低

- 拉取的"新闻"实为行情板块资讯，非公司专项深度信息
- 阿里巴巴的新闻条目全部是南向资金流动，无一条涉及阿里本身

## 改进方案

### Layer 1：修复数据层 (`data/fetcher.py`)

**1-A 修复港股 K 线**

- 诊断 `fetch_daily_kline_hk` 实际失败原因（网络/接口变更），添加备用接口
- 在 `fetcher.py` 中扩充港股 K 线备用源（如 `stock_hk_hist_min_em` 降级到 `akshare` 其他港股接口）

**1-B 新增数据拉取函数**


| 函数名                            | AkShare 接口                                                | 补充什么                    |
| ------------------------------ | --------------------------------------------------------- | ----------------------- |
| `fetch_pe_history(code)`       | `stock_a_indicator_lg` / `stock_hk_valuation_baidu`（历史序列） | PE/PB 近 3 年历史，用于计算当前分位数 |
| `fetch_shareholder_info(code)` | `stock_shareholder_fund_em` / `stock_zh_a_gdhs`           | 机构持仓、前十大股东、外资占比         |
| `fetch_margin_data(code)`      | `stock_margin_detail_szse`/`stock_margin_detail_sse`      | 融资融券余额趋势                |
| `fetch_north_flow(code)`       | `stock_em_hsgt_north_net_flow_in`                         | 北向资金净流入                 |
| `fetch_analyst_ratings(code)`  | `stock_analyst_rating_em`                                 | 分析师评级与目标价               |
| `fetch_announcements(code)`    | `stock_notice_report`                                     | 重大公告（回购/减持/增持）          |
| `fetch_cash_flow(code)`        | `stock_cash_flow_sheet_by_yearly_em`                      | 经营/投资/筹资现金流             |


**1-C 扩展 `fetcher.py` 已有函数**

- `fetch_financial_summary` 补充 ROIC、自由现金流计算（营业现金流 - 资本支出）

### Layer 2：量化层对齐 (`analysis/quantitative.py`)

**2-A 重写 `score_growth`**

- 用财务摘要里的「营收同比(%)」和「净利同比(%)」替代价格涨幅
- 价格动量保留在 `score_momentum` 中（当前已有）

**2-B 重写 `score_profitability`**

- 使用 `ROE(%)`、`净利率(%)`、`毛利率(%)` 实际计算
- 构建多指标加权：ROE 权重 50%、净利率 30%、毛利率 20%

**2-C 新增 `score_valuation_percentile`**

- 利用拉取的 PE 历史序列，计算当前 PE 在近 3 年的百分位
- 分位越低得分越高（更便宜）

### Layer 3：Prompt 层补强

**3-A 升级 `context_builder.py`**
在 `build_context` 中新增数据段落：

```python
sections = [
    ...
    format_pe_percentile(pe_history),        # PE/PB 历史分位
    format_shareholder_info(shareholders),    # 股东结构
    format_margin_data(margin),               # 融资融券
    format_analyst_ratings(ratings),          # 分析师评级
    format_cash_flow_summary(cash_flow),      # 现金流摘要
    format_announcements(announcements),      # 近期重大公告
]
```

**3-B 新增 Prompt 模板**

- `prompts/research/06-capital-flow.md`：资金流向与机构动态分析（北向资金、融资融券、大股东行为）
- `prompts/research/07-catalyst.md`：近期催化剂追踪（公告解读、分析师调整、行业政策）

**3-C 调整 `orchestrator.py`**

- `list_agent_dimensions()` 自动扫描 `prompts/research/*.md`，新 Prompt 文件落地即可被识别，无需手动注册

## 改进后 context.md 结构（对比）

```
现在：基本信息 → 五维评分（大量50.0）→ 估值 → K线(N/A) → 技术面(N/A) → 财务摘要 → 新闻(泛化)

改后：基本信息 → 股东结构 → 五维评分(真实数据) → 估值+历史分位 → K线+技术面 
     → 财务摘要+现金流 → 融资融券趋势 → 北向/南向资金 → 分析师评级 → 重大公告 → 新闻
```
